# 景区导览服务 AI 数字人总体设计文档

## 1. 建设目标

项目面向灵山胜境游客和景区管理者，提供 7×24 小时可交互的 AI 数字人导游。
游客能够通过语音、文字和图片获得可靠讲解与个性化路线；管理者能够维护知识、
配置数字人并查看游客关注点、满意度和情感趋势。

## 2. 总体架构

```text
游客浏览器（HTTP/HTTPS）
  ├─ 文字 / 麦克风 / 图片 / 兴趣 / 位置
  └─ WebRTC 数字人音视频
             │
             ▼
FastAPI 导览网关（8001 / 8443）
  ├─ ASR：GLM-ASR
  ├─ RAG 服务（8020，仅内部）：BGE-M3 + FAISS + 有界会话历史
  ├─ LLM：GLM-4.7-Flash API / 本地 Qwen2-7B（流式增量输出）
  ├─ Vision：GLM-4V 识景后再次进行知识库校准
  ├─ TTS：GLM-TTS PCM 分块流式合成
  ├─ SQLite：交互日志、反馈、数字人配置
  └─ 管理后台：知识库、数字人、运营分析
             │
             ▼
LiveTalking（8010，仅内部）→ Wav2Lip FP16 → WebRTC
```

## 3. 功能设计

### 3.1 游客交互端

1. 文字与语音输入：浏览器录音转 16kHz WAV，调用 GLM-ASR，再进入统一问答链路。
2. 景区知识问答：回答必须基于检索资料，资料不足时明确拒绝编造。
3. 数字人回答：首个完整短句生成后立即进入 TTS 与口型流水线，不等待全文完成。
4. 个性化路线：按历史、自然、亲子兴趣返回不同路线、时长和讲解重点。
5. 多模态识景：上传或拍摄图片，由视觉模型识别，再用本地知识库校准讲解。
6. 弱定位：支持 GPS、二维码、Wi-Fi 节点和手动选择的降级定位接口。
7. 满意度反馈：采集 1–5 分及文字意见，进入情感趋势和服务建议。

### 3.2 管理后台

1. 数据概览：今日/近七日服务游客、累计问答、响应时间、满意度、热门咨询。
2. 知识库管理：上传或同名更新 DOCX/TXT/Markdown，删除管理员文档，自动重建向量索引。
3. 数字人管理：维护名称、形象、声音、服装风格和默认表情，新会话生效。
4. 感受度报告：正/中/负向趋势、近期提问、满意度记录和规则化运营建议。

## 4. 核心技术

### 4.1 可靠 RAG

- 官方 DOCX、结构化 DOCX、XLSX 和管理员资料按标题、段落切分，长片段保留句级重叠。
- BGE-M3 生成归一化向量，FAISS `IndexFlatIP` 执行余弦相似度检索；命中景点名或别名时先按景区元数据缩小候选集。
- RAG 作为单独的本机服务持有模型和索引，多个游客/管理 API 监听器无需重复加载 BGE-M3。
- 系统提示词约束数字、日期、重量等事实必须来自检索上下文。
- 流式响应先返回稳定引用 ID，再输出文本增量；回答可直接进入按句 TTS 与 LiveTalking。
- 不提供会话 ID 时自动创建新 ID；后续携带相同 ID 可理解代词追问。历史受 TTL、会话数和轮数限制，取消或失败的半截回答不入库。

### 4.2 低延迟语音数字人

系统采用 ASR → LLM 首句 → TTS PCM 分块 → Wav2Lip 的流水线。LLM 输出过程中按
句号、问号和软长度边界切句；第一个短句可立即播报。Wav2Lip 使用 FP16 和 batch 8，
在保证口型表现的同时降低显存占用。

### 4.3 游客洞察

聊天记录按会话和时间保存。情感分析融合反馈评分与游客文本关键词，生成近七日服务
趋势、关注主题和服务建议。该模块可进一步替换为专用多模态情感模型。

## 5. 数据设计

- `chat_logs`：会话、角色、内容、检索/情感/延迟元数据、时间。
- `feedback`：会话、1–5 分、意见、情感类别、时间。
- `avatar_settings`：数字人名称、模型、声音、服装、表情、更新时间。
- `llm/index/faiss.index`：BGE-M3 向量索引。
- `llm/index/metadata.json`：与向量逐条对应的知识片段、来源、景区和小节元数据。
- RAG 会话：进程内 TTL/LRU 有界存储；服务重启即清空，不保存游客长期画像。

## 6. 非功能性设计

- 准确性：15 道标准事实题实测 100%，超过赛题要求的 90%。
- 延迟：语音结束至数字人首个非静音音频帧实测 3373ms，低于 5 秒目标。
- 稳定性：LiveTalking 不可用时游客端自动回退 Live2D，并每 10 秒重连。
- 可维护性：API、RAG、数字人和静态前端解耦；所有模型及端口可通过环境变量调整。
- 安全性：模型密钥仅保存在服务端；RAG/LiveTalking 只监听内部端口，索引重建和
  知识文档修改只允许登录后的独立管理端口访问；XMOV 浏览器密钥默认不下发。

## 7. 创新点

1. “首句即说”的跨模型流水线，把等待全文的串行链路改造成按句并行链路。
2. 视觉识别结果再次经过本地知识库校准，减少通用视觉模型的景点幻觉。
3. 网络定位失效时提供二维码、Wi-Fi 和手动选择的多级降级策略。
4. 游客交互、数字人表现和管理洞察形成数据闭环，而非单一聊天演示。

## 8. 当前边界与演进

- 当前提供一个 Wav2Lip 真人头像；可通过管理端和头像生成接口扩充形象。
- GPS/Wi-Fi 为可替换的演示适配层，生产部署需接入真实坐标和锐捷网络位置数据。
- 当前情感分析以轻量规则保证本地可用，后续可接入表情、声学特征和专用模型。
