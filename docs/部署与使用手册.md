# 部署与使用手册

## 1. 环境

- Linux、NVIDIA GPU、CUDA 12.x
- Conda 环境：`ccc`（导览 API/LiveTalking）和 `softcup`（BGE-M3/FAISS RAG）
- Python 3.12
- 可用的智谱 API Key，存放在根目录 `softcup_glmkey` 或环境变量 `ZHIPU_API_KEY`
- 下载模型时可使用 `http://127.0.0.1:7890` 代理；运行期访问本地服务必须设置 `NO_PROXY`

## 2. 启动服务

```bash
source /home/anaconda/etc/profile.d/conda.sh
conda activate ccc
cd /home/gmn/codes/cup

# 选择空闲物理 GPU。若 GPU 1 被占用，可改为 2。
LIVETALKING_GPU=2 bash deploy/start_livetalking.sh
bash deploy/start_api.sh
```

端口说明：

| 端口 | 用途 | 是否直接提供给游客 |
|---|---|---|
| 8001 | HTTP 游客端、开放 API | 是，文字功能可用 |
| 8443 | HTTPS + TURN/TCP，游客页面及 WebRTC 媒体中继 | 是，麦克风和远程访问推荐 |
| 8010 | LiveTalking WebRTC/口型内部服务 | 否，由 8001/8443 代理 |
| 8020 | BGE-M3 + FAISS + GLM RAG 服务 | 否，仅监听 127.0.0.1 |
| 8444 | 独立 HTTPS 管理后台 | 是，仅供管理员 |

## 3. 使用入口

- 游客端：`http://192.168.200.27:8001/`
- 麦克风端：`https://192.168.200.27:8443/`
- 管理端：`https://192.168.200.27:8444/admin`
- API 文档：`http://192.168.200.27:8001/docs`

HTTPS 首次打开需要接受自签名证书。浏览器如果运行在其他电脑，`localhost` 指向那台
电脑自身；可使用服务器 IP，或仅转发 8443。服务会在同一端口自动区分 HTTPS 与
TURN/TCP，因此远程浏览器的 LiveTalking 媒体也能通过 TCP 隧道。

## 4. 游客端操作

1. 选择历史文化、自然风光或亲子家庭兴趣。
2. 输入问题后选择“文字问答”，或在 HTTPS 页面点击“点击说话”。
3. 可上传音频、拍照识景、请求当前位置讲解和个性化路线。
4. 数字人会在首句生成后开始播报；再次提问会打断上一段播报。
5. 游览结束可提交 1–5 分评价和文字建议。

## 5. 管理端操作

- 数据概览：查看人次、问答、延迟、情感和热门景点。
- 知识库：上传 `.docx`、`.txt`、`.md`；同名文件视为更新；官方资料只读。
- 数字人：修改名称、已安装头像、TTS 声音、服装描述和默认表情。
- 感受报告：查看近期问题、评价与系统生成的服务改进建议。

管理端使用独立端口和服务端签名的 HttpOnly/Secure 会话 Cookie。公网部署前必须
修改默认管理员密码，并配置固定的 `ADMIN_SESSION_SECRET`；知识文档修改和索引重建
不能从游客端口调用。

## 6. 健康检查与测试

```bash
curl http://127.0.0.1:8001/health
curl http://127.0.0.1:8020/health
curl http://127.0.0.1:8001/v1/livetalking/status

cd /home/gmn/codes/cup/services/api
python scripts/eval_accuracy.py
```

预期准确率不低于 90%。当前实测为 100%。

## 7. 日志与排障

- API：`deploy/api.log`
- HTTPS API：`deploy/api-ssl.log`
- RAG：`deploy/rag.log`
- RAG 健康看护：`deploy/rag-watchdog.log`
- 数字人：`deploy/livetalking/service.log`

常见问题：

- `403 Forbidden`：不要访问 8010 的目录；产品入口是 8001 或 8443。
- `ERR_EMPTY_RESPONSE`：对应端口没有监听，先检查 `ss -lntp | rg '8001|8443|8010|8020'`。
- RAG 首次启动失败：检查 `deploy/rag.log`、BGE-M3 本地缓存和 `RAG_GPU`；已有模型缓存时保持 `RAG_HF_OFFLINE=true`。
- `CUDA out of memory`：用 `nvidia-smi` 选择空闲 GPU，再设置 `LIVETALKING_GPU` 或 `RAG_GPU`。
- 麦克风不可用：浏览器要求 HTTPS 或 localhost，请使用 8443。
- 数字人无视频：确认 `/v1/livetalking/status` 中 `ready=true`，再检查浏览器 WebRTC。
