# 产品方案介绍 PPT 大纲

建议 12 页，答辩 8–10 分钟。

1. **封面**：灵山小向导——景区导览服务 AI 数字人；团队、学校、赛题编号 A5。
2. **行业背景与痛点**：旺季导游稀缺、录音导览单向、缺乏情感连接、管理反馈盲区。
3. **产品目标**：一个游客交互端 + 一个景区运营后台 + 一条持续优化的数据闭环。
4. **游客体验旅程**：入园选兴趣 → 语音提问 → 数字人讲解 → 拍照识景 → 路线推荐 → 评价。
5. **总体架构**：浏览器、FastAPI 网关、RAG/GLM、多模态模型、LiveTalking、SQLite 数据层。
6. **可靠知识问答**：本地资料切片、混合检索、引用追踪、拒答约束；展示 15/15 准确率。
7. **自然语音数字人**：ASR→语义句→完整语义音频→Wav2Lip；展示自然断句、完整播报和延迟实测。
8. **多模态与个性化**：语音/文字/图片/情绪、历史/自然/亲子路线、弱定位降级。
9. **管理运营闭环**：知识库维护、数字人配置、热门问答、情感趋势、满意度和建议。
10. **创新亮点**：首句即说、视觉结果 RAG 校准、位置多级降级、Live2D 自动回退。
11. **测试与工程质量**：100% 标准题、低于 5 秒、服务健康检查、后台托管、密钥不下发。
12. **应用与规划**：接入真实 GPS/Wi-Fi、更多景区/方言/头像、边云协同、商业推广。

答辩时应现场打开游客端与管理端，避免只放录屏。所有性能数字注明测试环境和时间。
